# モデルルーティング & サブエージェント委譲ルール

トークン（＝レート制限枠）効率のためのルール。**サブエージェントは fresh context で起動するため
再読オーバーヘッドがある。委譲は「main context の汚染回避」がそれを上回るときだけ行う。**

## 委譲する（サブエージェント向き）

| タスク | 使うエージェント |
|--------|----------------|
| 広域コード探索・シンボル検索（結果だけ欲しい） | 組み込み Explore または `explorer`（haiku） |
| 長いログ・大量ファイルの読解と要約 | `explorer`（haiku） |
| 機械的作業（rename、一括置換、テスト実行と結果報告） | `mechanic`（haiku） |
| 独立した並列調査（互いに依存しない複数トピック） | 並列サブエージェント |
| コードレビュー・セキュリティレビュー | `reviewer`（セッションモデルを継承） |

## 委譲しない（main で直接）

- ファイル数個の読み書きで済む編集・実装
- 会話・質問への回答・小さな調査
- すでに main context に読み込んだファイルへの作業（再読コストが無駄）

## モデル選択

- 設計判断・アーキテクチャ・難しいデバッグ: 最上位モデル（fable。使えなくなったら opus にフォールバック）
- 通常の実装: セッションのデフォルトモデル
- 機械的・定型作業: haiku で十分

## その他の効率ルール

- 長いセッションでは区切りのよいところで `/compact` を検討
- 不要な MCP サーバー・プラグインは有効化しない（context を常時圧迫するため）
